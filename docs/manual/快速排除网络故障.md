# 常见答疑&快速排除网络故障

---

## 一、网线已连接但无法连接网络

1. **检查路由器 WAN 口是否获得 IP 地址**  
   - 查看方法：进入路由器后台总览页面，查看 WAN 口（或外网接口）的状态。如果获得类似 `192.168.0.114` 等 IP 地址，则路由器正常工作。无法上网的原因可能是：
     - **未认证入网许可**：需要进入认证页面进行认证。
     - **IP 冲突**：路由器的局域网（LAN）IP 可能与 WAN 口冲突，需要更改路由器的 LAN 口 IP（可百度具体方法）。
     - **设备被拉黑**：需要申请解封或更改路由器 WAN 口的 MAC 地址（关于 MAC 地址的基础知识请先百度，群内也有相关插件）。

2. **如果路由器无法获得 IP 地址**  
   - 可能是以下原因：
     - 网线未插好。
     - 网线或接口（路由器或墙上的）损坏。
   - 排查后更换损坏的硬件。

3. **代理软件异常**  
   - 检查手机或电脑的代理设置，重启 VPN 或代理软件，并确保关闭代理功能。

4. **服务商网络故障**  
   - 检查周围人的网络是否正常。如果服务商网络出现问题，需及时报修。

5. **因路由器代理被封禁**  
   - 如果通过路由器代理上网导致服务商封禁账号，会出现莫名其妙断网的情况（部分服务商不会提醒账号被封禁）。  
   - 解决方法：更改路由器 WAN 口的 MAC 地址，重启接口并重新认证。

---

## 二、通过路由器无法进入认证页面（Web 网页认证类）

1. **确保路由器能获取到 IP 地址**  
   - 排除第一类问题，保证路由器正常连接网络。

2. **关闭“重绑定保护”**  
   - 刷入 OpenWRT 的路由器可尝试在“网络-DNS”选项菜单中关闭“重绑定保护”选项，然后重启设备再尝试认证。

3. **认证页面无法自动重定向**  
   - 手动复制认证页面地址，在浏览器中直接打开。

4. **MAC 地址被服务商拉黑**  
   - 更改 WAN 口 MAC 地址后重新认证。

---

## 三、路由器突然进不去后台了？

1. **电源问题**  
   - 检查路由器指示灯是否正常，确保电源正常工作。

2. **固件或硬件问题**  
   - 重刷系统或恢复出厂设置，排除固件问题后
3. **系统崩溃**  
   - 重启路由器。如果问题仍未解决，则尝试重刷固件或恢复出厂设置。若无法刷入系统，可能已变砖，需维修或更换路由器。

4. **网络接口冲突**  
   - 如果电脑同时使用多个网络接口（如 WiFi 和网线），可能优先使用了错误的接口，导致无法进入后台。解决方法：断开其他接口的网络，重新尝试登录后台。

5. **客户端使用代理软件（VPN）导致解析异常**
   - 关闭VPN软件，或重置系统代理。

## 五、网速慢或网络卡顿怎么办？

1. **运营商限速**  
   - 购买更高规格套餐。

2. **路由器性能限制**  
   - 公版驱动速率有限，如果运营商网速很快（如千兆网），低端路由器可能跑不满。  
   - 软件对系统负载影响较大，也可能导致收发速率受限（低端路由器较常见）。  
   - 解决方法：尝试更换插件或升级硬件。

3. **网络被占用**  
   - 如果有设备在大量下载，建议使用限速插件 `luci-app-qos` 等进行管理。

4. **多播叠加**  
   - 尝试通过多播叠加提高网速，具体方法可自行查阅相关资料。
